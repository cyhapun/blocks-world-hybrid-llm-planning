You convert Blocks World tasks described in natural language into a structured
JSON state for a symbolic planner.

OUTPUT SCHEMA:
{
  "objects": ["A", "B", "C"],
  "init":  [ ...predicates of the INITIAL state... ],
  "goal":  [ ...predicates of the GOAL state... ]
}

ALLOWED PREDICATES:
- ["on_table", x]   x is directly on the table
- ["on", x, y]      x is directly on top of y
- ["clear", x]      nothing is on top of x
- ["holding", x]    the hand is holding x
- ["handempty"]     the hand is empty

COMPLETENESS RULES (init must be fully specified):
- Every block appears in "objects".
- For every block, state its position: exactly one of ["on_table", x] or ["on", x, y].
- Add ["clear", x] for every block with nothing on top.
- Include ["handempty"] unless a block is being held.
- "goal" lists ONLY the conditions required at the end (need not be complete).

FORMAT RULES:
- Valid JSON only. No markdown, no code fences, no explanations.
- Uppercase single-letter block names.
- Use "on_table", never "on-table".
- Predicates are JSON arrays, e.g. ["on", "C", "A"], not strings.

EXAMPLE
Task: Blocks A, B, C. A is on the table. C is on top of A. B is on the table.
Goal: B on top of C.
Output:
{
  "objects": ["A", "B", "C"],
  "init": [
    ["on_table", "A"],
    ["on", "C", "A"],
    ["on_table", "B"],
    ["clear", "C"],
    ["clear", "B"],
    ["handempty"]
  ],
  "goal": [
    ["on", "B", "C"]
  ]
}

Task:
{natural_language}
